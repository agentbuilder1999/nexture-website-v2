## 一、TheraSeus™ 功能映射到技术需求

### 1.1 图像过滤（核心功能：移除 90% 无临床意义图片）

**临床背景**: 
- 胶囊内镜单次检查产生 50,000-100,000 张图像
- 其中约 80-90% 是无临床意义的正常黏膜、模糊帧、肠道准备液
- 医生目前需手动浏览全部图像，耗时 60-120 分钟

**技术方案**:
| 过滤层 | 方法 | 目的 |
|--------|------|------|
| L1: 帧质量 | Laplacian 方差 + BRISQUE | 移除模糊/过暗/过曝帧 |
| L2: 重复帧 | pHash 感知哈希 + 光流法 | 移除连续相似帧（蠕动静止段）|
| L3: 内容分类 | EfficientNet-B3 / DINOv2 | 分类为：正常黏膜 / 肠液 / 气泡 / 有临床意义 |
| L4: 置信阈值 | 可调阈值 | 保守模式（少过滤）vs 激进模式（多过滤），医生可调 |

**验收标准（来自官网）**:
- 过滤率 ≥ 90%（50,000 → ≤5,000 张）
- 假阴性率 < 1%（有意义图片不能被误删）
- 处理时间 < 5 分钟 / 全量图像

### 1.2 AI 检测（出血和异常病变）

**临床需要检测的病变类型**:
| 类型 | 发生率 | 临床紧急度 | 检测难度 |
|------|--------|-----------|---------|
| 活动性出血 | 5-10% | 🔴 高 | 低（颜色特征明显）|
| 血管扩张 | 3-5% | 🟡 中 | 中 |
| 息肉 | 2-5% | 🟡 中 | 中高（形态多样）|
| 溃疡/糜烂 | 5-15% | 🟡 中 | 中 |
| 肿瘤/肿块 | <1% | 🔴 高 | 高（稀有样本）|
| Crohn 病变 | 变化大 | 🟡 中 | 中高 |
| 寄生虫 | <1% | 🟢 低 | 低 |

**技术方案**:
- **检测**: YOLOv8-x（高精度 mAP ~54%）对过滤后图像做病灶定位
- **超分预处理**: Real-ESRGAN 将 512×512 → 1024×1024，提升小病灶识别率
- **分割**: SAM2 对检测到的病灶做精细边界分割
- **分类**: EfficientNet + 自训练权重，区分病变类型和严重程度

**验收标准**:
- 出血检测灵敏度 ≥ 95%
- 息肉检测灵敏度 ≥ 90%
- 假阳性率 < 5%（避免过多误报增加医生负担）

### 1.3 结构化报告

**官网描述**: "Streamlined documentation and improved communication with auto-compiled key findings"

**技术方案**:
| 报告组件 | 数据来源 | 生成方式 |
|----------|---------|---------|
| 患者信息 | 输入/导入 | 模板填充 |
| 检查概要 | 图像统计 | 自动统计（总帧数/过滤数/检出数）|
| 异常发现列表 | AI 检测结果 | 按位置+类型+严重度排序 |
| 代表性图片 | 检测置信度 Top-N | 自动筛选 + 医生确认 |
| 位置映射 | 帧时间戳 → 解剖位置估算 | 基于通过时间的粗略定位 |
| 诊断建议 | 检测结果 + LLM | Qwen2.5 或云端 LLM 生成草稿 |
| PDF 输出 | 上述汇总 | reportlab / fpdf2 |

**验收标准**:
- 报告生成时间 < 30 秒
- 包含全部异常发现 + 代表图片
- 输出 PDF 格式，兼容 EMR（电子病历）系统导入

### 1.4 部署架构（Cloud + On-premise）

**官网**: "Cloud with HIPAA or on-premise options available"

```
Cloud 部署：
  客户端（Web/Desktop）
    → HTTPS → API Gateway（AWS ALB）
    → FastAPI 后端
    → GPU Worker（推理）
    → S3（图像存储，加密）
    → RDS/DynamoDB（结构化数据）
    → 全链路 HIPAA 合规

On-premise 部署：
  Docker Compose / Kubernetes
    → 本地 GPU 服务器
    → 本地文件存储（加密）
    → 隔离网络，无外部通信
```

## 三、胶囊内镜临床工作流（当前 vs TheraSeus）

### 3.1 当前工作流（无 AI）

```
1. 患者吞服胶囊（8-12小时记录）
2. 数据下载到工作站（~30min）
3. 医生打开阅片软件（RAPID/ES-View 等）
4. 手动浏览 50,000+ 张图像（60-120min）
5. 标记异常发现
6. 手动撰写报告（15-30min）
7. 与患者沟通结果

总时间：90-180 分钟 / 例
```

### 3.2 TheraSeus 工作流

```
1. 患者吞服胶囊（不变）
2. 数据上传到 TheraSeus（自动/拖放）
3. AI 处理（5min）:
   - 帧过滤 → 50,000 → 5,000
   - 超分辨率增强
   - 病灶检测 + 分割
   - 报告草稿生成
4. 医生审阅 AI 精选图像（6min）
5. 确认/修改报告（2min）
6. 一键导出 PDF + EMR 导入

总时间：13 分钟 / 例（节省 85-93%）
```

## 四、数据与隐私要求

### 4.1 法规清单

| 法规 | 适用地区 | 要求 |
|------|---------|------|
| **Medsafe SaMD** | NZ | AI 诊断软件需分类注册 |
| **HIPAA** | 美国 | PHI 加密+访问控制+审计日志 |
| **NZ Privacy Act 2020** | NZ | 个人健康信息保护原则 |
| **TGA** | 澳大利亚 | SaMD 分类（如出口澳洲）|
| **MDR/IVDR** | 欧盟 | CE 标记（如出口欧洲）|
| **FDA 510(k)/De Novo** | 美国 | AI/ML SaMD 预市场审查 |

### 4.2 数据安全基线要求

- 静态加密：AES-256（存储）
- 传输加密：TLS 1.3
- 访问控制：RBAC + MFA
- 审计日志：所有数据访问记录
- 数据匿名化：训练数据必须去标识化
- 数据保留：按当地法规保留+销毁策略
- 数据驻留：NZ/US 数据不出境（或明确同意）
